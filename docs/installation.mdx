# Installation

Set up the Arete CLI and SDKs for your project.

---

## CLI Installation

The Arete CLI (`a4`) manages stacks, deployments, and SDK generation.

### From Source (Recommended)

```bash
# Clone the repository
git clone https://github.com/HyperTekOrg/arete
cd arete-oss

# Install globally
cargo install --path cli

# Verify installation
a4 --version
```

### Build Without Installing

```bash
cargo build --release -p a4-cli
# Binary available at: target/release/a4
```

## SDK Installation

### TypeScript / React

```bash
npm install arete-react
```

**Peer dependencies:**
- `react` ^18.0.0
- `zustand` ^4.0.0

```bash
# Install peer dependencies if needed
npm install react zustand
```

### Rust

Add to your `Cargo.toml`:

```toml
[dependencies]
a4-sdk = "0.2"
tokio = { version = "1.0", features = ["full"] }
```

Quick example:

```rust
use a4_sdk::prelude::*;
use my_stack::{Token, TokenEntity};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let a4 = Arete::connect("wss://ore.stack.arete.run").await?;
    
    // List all entities
    let tokens = a4.list::<TokenEntity>().await;
    
    // Watch for live updates (lazy - no await needed)
    let mut stream = a4.watch::<TokenEntity>()
        .filter(|u| u.data().map(|t| t.volume > 1000).unwrap_or(false));
    
    while let Some(update) = stream.next().await {
        println!("Update: {}", update.key());
    }
    
    Ok(())
}
```

See the [Rust SDK README](https://github.com/HyperTekOrg/arete/tree/main/rust/a4-sdk) for full documentation.

## Authentication

Before deploying stacks, authenticate with Arete:

```bash
# Create a new account
a4 auth register

# Or login to existing account
a4 auth login

# Verify authentication
a4 auth whoami
```

Credentials are stored in `~/.arete/credentials.toml`.

## Project Setup

### Initialize a New Project

```bash
# In your project directory
a4 init
```

This creates `arete.toml` and auto-discovers any existing AST files.

### Configuration File

**arete.toml:**

```toml
[project]
name = "my-project"

[sdk]
output_dir = "./generated"

# Stacks are auto-discovered from .arete/*.ast.json
# Define explicitly for custom naming:
[[stacks]]
name = "my-stack"
ast = "MyEntity"
```

### Validate Configuration

```bash
a4 config validate
```

## Development Setup

### For Stack Development (Rust)

If you're writing stacks, add these dependencies:

```toml
[dependencies]
arete = { path = "../arete-oss/main/arete", features = ["full"] }
a4-macros = { path = "../arete-oss/main/a4-macros" }
a4-interpreter = { path = "../arete-oss/main/interpreter" }

# For local server
a4-server = { path = "../arete-oss/main/rust/a4-server" }

# Required
tokio = { version = "1.0", features = ["full"] }
serde = { version = "1.0", features = ["derive"] }
borsh = { version = "1.5", features = ["derive"] }
```

### For Frontend Development

```bash
# Install SDK
npm install arete-react

# For local development, use localhost WebSocket
# For production, use your deployed stack URL
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `a4: command not found` | Ensure `~/.cargo/bin` is in your PATH |
| `Cannot find module 'arete-react'` | Run `npm install arete-react` |
| `Peer dependency warnings` | Install `react` and `zustand` explicitly |
| `Authentication failed` | Run `a4 auth login` to refresh credentials |
