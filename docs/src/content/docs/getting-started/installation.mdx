## CLI Installation

The Hyperstack CLI (`hs`) manages stacks, deployments, and SDK generation.

### From Source (Recommended)

```bash
# Clone the repository
git clone https://github.com/HyperTekOrg/hyperstack-oss
cd hyperstack-oss

# Install globally
cargo install --path cli

# Verify installation
hs --version
```

### Build Without Installing

```bash
cargo build --release -p hyperstack-cli
# Binary available at: target/release/hs
```

## SDK Installation

### TypeScript (Core)

For non-React environments (Node.js, Vue, Svelte, vanilla JS):

```bash
npm install hyperstack-typescript
```

No peer dependencies required. Framework-agnostic with AsyncIterable-based streaming.

### TypeScript / React

For React applications:

```bash
npm install hyperstack-react
```

**Peer dependencies:**

- `react` ^19.0.0
- `zustand` ^4.0.0

```bash
# Install peer dependencies if needed
npm install react zustand
```

:::note[Which package should I use?]

- **React/Next.js apps** → `hyperstack-react` (includes hooks, providers, automatic re-rendering)
- **Everything else** → `hyperstack-typescript` (pure TypeScript, framework-agnostic)
  :::

### Rust

Add to your `Cargo.toml`:

```toml
[dependencies]
hyperstack-sdk = { git = "https://github.com/HyperTekOrg/hyperstack-oss", branch = "main" }
tokio = { version = "1.0", features = ["full"] }
```

### Python

```bash
pip install hyperstack-sdk
```

## Authentication

Before deploying stacks, authenticate with Hyperstack:

```bash
# Create a new account
hs auth register

# Or login to existing account
hs auth login

# Verify authentication
hs auth whoami
```

Credentials are stored in `~/.hyperstack/credentials.toml`.

## Project Setup

### Initialize a New Project

```bash
# In your project directory
hs init
```

This creates `hyperstack.toml` and auto-discovers any existing AST files.

### Configuration File

**hyperstack.toml:**

```toml
[project]
name = "my-project"

[sdk]
output_dir = "./generated"

# Stacks are auto-discovered from .hyperstack/*.ast.json
# Define explicitly for custom naming:
[[stacks]]
name = "my-stack"
ast = "MyEntity"
```

### Validate Configuration

```bash
hs config validate
```

## Development Setup

### For Stack Development (Rust)

If you're writing stacks, add these dependencies:

```toml
[dependencies]
hyperstack = { path = "../hyperstack-oss/main/hyperstack", features = ["full"] }
hyperstack-macros = { path = "../hyperstack-oss/main/hyperstack-macros" }
hyperstack-interpreter = { path = "../hyperstack-oss/main/interpreter" }

# For local server
hyperstack-server = { path = "../hyperstack-oss/main/rust/hyperstack-server" }

# Required
tokio = { version = "1.0", features = ["full"] }
serde = { version = "1.0", features = ["derive"] }
borsh = { version = "1.5", features = ["derive"] }
```

### For Frontend Development

```bash
# Install SDK
npm install hyperstack-react

# For local development, use localhost WebSocket
# For production, use your deployed stack URL
```

// If this compiles, you're ready to go

## Troubleshooting

| Issue                                        | Solution                                   |
| -------------------------------------------- | ------------------------------------------ |
| `hs: command not found`                      | Ensure `~/.cargo/bin` is in your PATH      |
| `Cannot find module 'hyperstack-react'`      | Run `npm install hyperstack-react`         |
| `Cannot find module 'hyperstack-typescript'` | Run `npm install hyperstack-typescript`    |
| `Peer dependency warnings`                   | Install `react` and `zustand` explicitly   |
| `Authentication failed`                      | Run `hs auth login` to refresh credentials |
